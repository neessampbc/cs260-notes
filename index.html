Exam scope (by topic)

Assembly (light)

  • Know enough to write small fragments (for example, three lines exercising a specific instruction or architecture feature).
  • You will not be asked to write a full program.

Hardware elements (160 review)

  • Know all hardware elements: MUXes, memory (read/write), register files, and so on.
  • Given a simple circuit, be able to design it.

Single cycle

  • Implement a given instruction on the single-cycle datapath (data + control path).
  • Why is single cycle good? Why is it bad?
  • How to determine the critical path.
  • How to calculate clock rate from the critical path.
  • When is single cycle better vs. worse than multicycle, pipelined, or microcoded?

Multicycle

  • How to write RTL.
  • Implement a given instruction on the multicycle datapath.
  • Identify the clock cycle time and MIPS rating.
  • When is multicycle better vs. worse than single cycle?

Microcode

  • Know the basic concept and how it differs from multicycle.
  • Know advantages and disadvantages (conceptual, not memorized formats).
  • Shankar will provide the format if needed; you may be asked to write one micro-instruction.

Pipelining

  • Throughput vs. latency — know what each means.
  • The three types of hazards: structural, data, and control.
  • Given a program, identify all hazards.
  • How data hazards are handled: forwarding and stalling.
  • Given a forwarding or stalling condition with a bug, identify the bug.
  • Given a modified architecture, interpret what its forwarding/stalling condition does.
  • Control hazards: branch prediction.
  • 1-bit branch predictor, 2-bit branch predictor, static branch predictor — know all three.
  • Tournament predictor: know roughly what it is (not in depth).
  • Clock rate for a pipeline = determined by the critical stage (not critical path).
  • Given a custom pipeline with its own specs, calculate timing and cycle count for a program.
  • When is pipelining better vs. worse than single cycle, multicycle, or microcoded?

Interrupts / exceptions

  • Know what an interrupt is.
  • Given a scenario, recognize that interrupts or exceptions are the right solution.

Fixed and floating point

  • Convert numbers to fixed and floating point.
  • Know the issues and limitations of both (rounding, overflow, precision, and so on).
  • Do not memorize IEEE 754 — Shankar will provide the format or invent one for the test.

Modern CPU architectures (brief)

  • You may see a picture and a question — know the rough idea.

How the exam is split

Part A · May 19 · 75 min

Everything above except two design questions

Big-picture comparisons, performance analysis, hazard identification, branch prediction, floating point, pipelining cycle counts, interrupt concepts.

Part B · May 26 · ~60 min

Two design questions only

  • Q1: Implement a given instruction on the single-cycle datapath (data + control).
  • Q2: Implement a given instruction on the multicycle datapath.
