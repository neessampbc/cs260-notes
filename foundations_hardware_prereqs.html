The Three Hardware Element Classes
hw.pdf Slide 5 · App. A.1 · "What are the building blocks?"

Every single component in a RISC-V processor falls into exactly one of three classes. Getting these categories straight is the prerequisite for understanding why the datapath looks the way it does.

hw.pdf — Slide 5 · Classes of Hardware Elements3/10
Shankar hw.pdf Slide 5 — reproduced. The orange = sequential is the key distinction for diagram notation.
Combinational elements are not clocked. Sequential elements are almost always clocked — writes happen on the rising clock edge. This is the course convention. Shankar draws sequential elements with diagonal hash marks through the box; combinational elements are plain boxes or triangles. You will be asked to identify which is which.
This is the course's framing question from day 1 of CPU design: "What are the primitive hardware elements?" (160's job). "How do we use them to build a processor?" (260's job). Every datapath question reduces to identifying which elements are present and whether they're combinational or sequential.

§ 02
Primitive Combinational Elements — Gates
hw.pdf Slides 6–9 · App. A.2 · "Built from transistors"
hw.pdf

Gates are the transistor-level building blocks. Every other combinational element (MUX, ALU, adder) is built from gates. You need to recognize the symbol, know the truth table, and know the key fact about NAND completeness.

Key fact from hw.pdf Slide 6: With n bits you can represent 2n values. Conversely, you need n bits to pick between 2n options. (More precisely: ⌈log₂(k)⌉ bits for k options.) You cannot connect the outputs of two logic units together — unless you like to see smoke.

§ 03
Non-Primitive Combinational Elements
hw.pdf Slides 10–12 · App. A.3 · MUX · Adder · ALU
hw.pdf

The datapath uses three non-primitive combinational elements constantly: MUX, Adder, and ALU. Every single-cycle diagram you've seen is just these elements wired together with registers.

Multiplexor (MUX)

A MUX routes one of its N inputs to the output based on a select signal. If SEL=0, output = In0. If SEL=1, output = In1. For a 4:1 MUX you need SEL to be 2 bits wide (2 bits → 4 options).

The SEL line width is ⌈log₂(N)⌉ bits for an N:1 MUX. A 4:1 MUX needs a 2-bit SEL. This directly maps to the hw.pdf "n bits for 2ⁿ options" fact. Exam questions often ask how wide the control signals on MUXes need to be.

The ALU is a combinational element that performs one of several operations on two inputs X and Y, selected by ALUOp. It also outputs a Zero flag (1 if F=0) used for beq.

The ALU Zero output is critical for beq. The control unit ANDs the Branch signal with ALU Zero — only when both are 1 does the PC take the branch target. § 04
Sequential Elements — Registers, Memory, Register File
hw.pdf Slides 13–16 · App. A.7–A.8 · Clocked storage
hw.pdf

Sequential elements remember a value across clock cycles. Unlike combinational elements, they don't instantly recompute — they hold their state until the rising clock edge triggers an update.

hw.pdf Slide 13 — How to use a sequential element:
Read location A: (1) Send A to the address input (assert Read=1 if needed). (2) Read contents from the data output.
Write X to location A: (1) Send A to address and X to write-data input. (2) Assert Write=1 on the write control signal. The value latches on the rising clock edge.

Clock Edge Timing — How Sequential Elements Update

PC is a 32-bit sequential register. It holds its value for an entire clock cycle. On each rising edge it latches the new value (PC+4 or branch target).
① REGISTER (hw.pdf Slide 14)

On rising clock edge, stores X → Y stays unchanged between edges. Width = number of bits (e.g., 32-bit for PC).

② MEMORY (hw.pdf Slide 15)

Characterized by words × bits per word. Read is combinational (output appears immediately). Write is clocked (needs Write=1 + rising edge).

③ REGISTER FILE (hw.pdf Slide 16)

32 registers × 32 bits. 2 read ports (combinational), 1 write port (clocked). Shankar's name: "called Registers by the text." 5-bit address selects register (2⁵=32).

Register File: 5-bit addresses (because log₂(32) = 5). Why 5 bits? From hw.pdf Fact 1 — you need 5 bits to pick between 32 registers. This connects directly to RISC-V instruction encoding: rs1, rs2, rd fields are all 5 bits wide (IR[19:15], IR[24:20], IR[11:7]).
A common exam mistake: saying the register file is combinational because "reads are instant." It is classified as sequential because it holds state and writes are clocked. The hatched box is the giveaway.

§ 05
Two Important Theorems
hw.pdf Slides 17–18 · "Why this all works"
hw.pdf
THEOREM 1 — NAND Completeness

Every combinational circuit can be built from just AND + OR gates (assuming inputs and their negations are available). Corollary: NAND gates alone suffice for any combinational circuit.

Why it matters: The entire ALU, MUX, ImmGen — every combinational element in the datapath — can in principle be built from NAND gates. Modern logic minimization (Karnaugh maps, Quine-McCluskey) finds the optimal such circuit.
THEOREM 2 — State Machine Decomposition

Every sequential circuit (state machine) can be decomposed into an output function and a next-state function. Each of those functions is just a combinational circuit.

Why it matters: The multicycle FSM you studied in HW4 is a state machine. Its "next state" logic (what state to go to) and "outputs" (which control signals to assert) are both combinational circuits driven by the current state + instruction opcode.

State Machine Structure (Theorem 2 Applied)

The multicycle FSM (HW4 / App. C) is exactly this structure. Current state + opcode → next state logic → state register updates on clock edge.
Theorem 2 is why Appendix C exists. The multicycle FSM's next-state and output functions are both combinational circuits. That's how you turn the FSM bubble diagram into actual hardware — each function becomes a truth table → a logic circuit.
§ 06
Introduction to Processor Design
§4.1–4.2 · Mar 31 lecture · Clocking methodology · Building a CPU from elements
§4.1–4.2

Now that you know the building blocks, §4.1–4.2 asks: how do we assemble them into a processor? The answer is two things: a clocking methodology and a design process.

The Clocking Methodology (§4.2)

The edge-triggered clocking methodology says: combinational logic computes during a clock cycle; sequential elements capture the result at the clock edge. This prevents feedback loops that would make the circuit unanalyzable.

One Clock Cycle in the Single-Cycle Datapath

During each cycle: sequential elements are read at the start, combinational logic computes the result, and the rising clock edge writes back to sequential elements. Everything happens in exactly one clock period.

The Two-Phase Design Process (§4.1)

PHASE 1 — Data Path

Wire up every component that data could ever flow through. Include every MUX, every path, every component — regardless of which instruction is running. Don't optimize yet.

→ Results in Figure 4.21 (the full datapath diagram)

PHASE 2 — Control Path

For each instruction type, specify which MUX selects are 0/1, which memory enables are asserted. The control unit decodes the 7-bit opcode and drives these signals.

→ Results in the control signal table (Figure 4.22)

Shankar's key quote from Apr 14: "The hard part is writing English. Hardware design is trivial." Design each instruction type in English first: what needs to be read, what computation happens, what gets written back. The hardware wiring follows directly from that description.

Shankar's RTL Design Method — Step by Step

For any instruction:
1. Write precise English — what does this instruction do?
2. Identify every sequential element mentioned
3. Draw a hatched box for each sequential element
4. Identify combinational elements needed (ALU, MUX, adder)
5. Connect — just re-read the English and draw the wires
6. For each MUX: which input should be selected? → that's your control signal

MOCK EXAM
Foundations — Exam-Style Questions

Harder than the practice quizzes. Try each before revealing. These are designed to predict Test 2 Part A patterns based on hw.pdf and Shankar's emphasis.

Q1 · Classification + Drawing Convention [5 pts]
For each component listed below, state whether it is combinational or sequential and describe how it would be drawn in Shankar's datapath diagrams. Then explain the significance of the drawing convention.
(a) ImmGen   (b) Data Memory   (c) PC+4 adder   (d) Register File (read side)   (e) AND gate for PCSrc
(a) ImmGen — Combinational. Plain box or oval. Takes instruction bits and produces a sign-extended immediate. No state.

(b) Data Memory — Sequential. Drawn with diagonal hatch marks. Holds state (the memory contents). Writes are clocked; reads are combinational but the element itself is classified as sequential.

(c) PC+4 Adder — Combinational. Plain triangle/arrow shape. Shankar explicitly said "boxes are combinational!" — the adder just adds 4 to its input with no stored state.

(d) Register File (read side) — Sequential (the whole element is sequential even though reads are combinational). Hatched box. Reading is combinational but the stored register values constitute state.

(e) AND gate — Combinational. Plain AND-gate symbol. Computes PCSrc = Branch AND Zero instantaneously.

Significance: The drawing convention tells you which elements update on the clock edge (hatched = must wait for rising edge to store a value). This determines what can change each cycle and sets up the critical path analysis.
Q2 · Bit-Width Calculation [4 pts]
A new ISA extension adds 12 ALU operations (beyond the original 4). How many bits must the ALUOp signal be? If a control unit decodes a 7-bit opcode, how many distinct instruction formats can it theoretically distinguish?
ALUOp width: Need to pick between 12 operations. ⌈log₂(12)⌉ = 4 bits (since 2³ = 8 < 12 ≤ 16 = 2⁴). A 3-bit signal would only support 8 operations — not enough.

Instruction formats from 7-bit opcode: 2⁷ = 128 distinct opcodes. The RISC-V spec uses a subset of these. This is why the opcode field being 7 bits gives plenty of room for extension.
Q3 · RTL Design [6 pts]
Write the English description of the sw rs2, offset(rs1) instruction's hardware operation, following Shankar's format. Then list: (a) which sequential elements are read, (b) which combinational elements are used, and (c) which sequential elements are written. Does this instruction write the register file?
English: "Send rs1 (IR[19:15]) and rs2 (IR[24:20]) from the register file. Sign-extend the S-type immediate from IR fields [31:25, 11:7]. Compute the address as rs1 + sign_ext(offset) using the ALU. Send rs2 as write data to data memory at that address. Assert MemWrite=1."

(a) Sequential elements read: Register file (reads rs1 → ReadData1, rs2 → ReadData2). Instruction memory (reads instruction).

(b) Combinational elements used: ImmGen (sign-extends S-type immediate), ALU (adds rs1 + offset, ALUSrc=1 selects immediate), ALUSrc MUX.

(c) Sequential elements written: Data Memory only (MemWrite=1, write address = ALU result, write data = rs2). PC is written with PC+4.

Does sw write the register file? No. RegWrite=0 for sw. The sw instruction moves data from a register to memory — no register receives a new value.
Q4 · State Machine Decomposition [5 pts]
The multicycle FSM from HW4 has 10 states. (a) How many flip-flops are needed to encode the current state? (b) Shankar's Theorem 2 says every sequential circuit decomposes into two functions. Name them and state what hardware implements each in the multicycle FSM.
(a) Flip-flops needed: ⌈log₂(10)⌉ = 4 bits (since 2³ = 8 < 10 ≤ 16 = 2⁴). So 4 D flip-flops are needed to encode 10 distinct states.

(b) Two functions from Theorem 2:
— Next-State Function: Takes current state + instruction opcode → outputs the next state encoding. Implemented as a combinational circuit (truth table → logic gates). In the FSM diagram this is the arc logic.
— Output Function: Takes current state (and possibly input) → outputs the control signals (RegWrite, MemRead, PCWrite, etc.). Also implemented as a combinational circuit. In the FSM diagram these are the signal labels inside each bubble.
Q5 · Clocking + Register File Design [4 pts]
An instruction simultaneously reads register x5 and writes register x5 in the same cycle (e.g., add x5, x5, x6). Under edge-triggered clocking, what value does the instruction read from x5 — the old value or the new value? Explain using the clock model.
It reads the old value (the value stored before the clock edge).

Under edge-triggered clocking: reads from sequential elements are combinational and happen at the start of the cycle (before the clock edge). Writes latch at the rising clock edge at the end of the cycle. So in add x5, x5, x6: the register file outputs the old x5 value at the beginning of the cycle, the ALU computes old_x5 + x6 during the cycle, and only at the rising edge does the new value get written back to x5. The read and write cannot collide because they occur at different points in the clock cycle.
